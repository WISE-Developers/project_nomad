## Technical Context for Next Session

### Critical Scope Decisions Made

For the 2-month timeline, we scoped MVP to **absolute minimum:**

**✅ In Scope:**
- Point ignition only (no polygons/lines)
- Single duration output (final day only)
- 50% probability contour display (not full raster)
- Automatic weather (SpotWX + FWI calc)
- Basic functional UI (not polished)
- Launch from EM3 via URL params
- FireSTARR deterministic models only

**❌ Out of Scope:**
- Perimeter ignitions
- Multiple output time steps
- Probability raster visualization
- WISE engine (FireSTARR only)
- Probabilistic models
- Export workflow
- Model history/review
- Mobile/PWA features
- Role-based permissions
- Configuration system (hardcode for single agency)

### Key Implementation Requirements

**Backend Must Handle:**
1. **Job orchestration:**
   - Accept job request (lat, lon, date, duration)
   - Validate ignition point (sample fuel grid, check for water/non-fuel)
   - Create job directory: `/appl/data/sims/job_${timestamp}_${id}/`

2. **Weather data pipeline:**
   - Fetch hourly forecast from SpotWX API
   - Fetch previous day FWI from CWFIS API (or use defaults)
   - Calculate FWI indices for each forecast hour
   - Generate weather CSV in FireSTARR format

3. **FireSTARR execution:**
   - Write weather CSV to job directory
   - Execute via `docker compose run` with proper arguments
   - Monitor completion (check for "Total simulation time was" in log)
   - Handle errors and timeouts

4. **Output processing:**
   - Convert UTM TIF → WGS84 TIF (via `gdalwarp`)
   - Generate 50% probability contour as GeoJSON (via `gdal_contour`)
   - Store result geometry in PostGIS
   - Serve GeoJSON to frontend

5. **Status tracking:**
   - Job queue with status (queued, running, completed, failed)
   - Status polling endpoint for frontend
   - Progress updates (if feasible)

**Frontend Must Provide:**
1. MapBox GL map with point placement
2. Simple wizard: location → date/time → duration → submit
3. Status polling during execution
4. Display fire perimeter on map when complete
5. Accept launch from EM3 via URL query params

### Data & APIs Needed

**SpotWX API:**
- Purpose: Hourly weather forecasts
- Status: Franco has access, needs API key details
- Usage: Fetch temp, RH, wind speed/direction, precipitation

**CWFIS API:**
- Purpose: Previous day FWI initialization values
- Status: Unknown if Franco has access
- Alternative: Use seasonal defaults (less accurate)

**FWI Calculation:**
- Options: Python `cffdrs` package, or JavaScript implementation
- Required: Calculate FFMC, DMC, DC, ISI, BUI, FWI from raw weather
- Critical: Must match Canadian Forest Fire Danger Rating System

**Fuel/DEM Grids:**
- Location: `/appl/data/generated/grid/100m/default/`
- Coverage: UTM zones 7-22 (full Canada)
- Format: GeoTIFF, 100m resolution, UTM NAD83
- Zones for NWT: 10, 11, 12
